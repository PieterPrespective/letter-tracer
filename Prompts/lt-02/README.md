# lt-02 — Pronunciation, word images & dark mode

Second round of features, on top of the shipped **lt-01** app (milestones
M0–M5: engine, base stroke data, home + feedback, words/sums, content editor +
IndexedDB, installable offline PWA).

This folder is **plans only** — no code yet — mirroring the `Prompts/lt-01`
style so the work can be picked up in small, reviewable PRs.

## Requested features

1. **Audio pronunciation** — a button on letters, numbers and words that plays
   the Dutch pronunciation. → [`01-audio-pronunciation.md`](./01-audio-pronunciation.md)
2. **Word images** — for words, show a background image of what the word means.
   → [`02-word-images.md`](./02-word-images.md)
3. **Dark mode** — proper dark-mode support (the Tab S8 uses a system dark
   theme). → [`03-dark-mode.md`](./03-dark-mode.md)

## Current baseline (what these build on)

- Content model in `src/model/types.ts`; base pack generated by
  `scripts/build-glyphs.ts` into `src/data/base-content.json`; user content in
  IndexedDB via `src/storage/*` and merged by `src/model/content.ts`.
- Trace screen `src/ui/screens/trace.ts`; home `src/ui/screens/home.ts`; editor
  `src/ui/screens/editor.ts`.
- Canvas rendering in `src/render/glyph-renderer.ts` (a hardcoded `COLORS`
  object — relevant to dark mode).
- Sound effects synthesised in `src/util/audio.ts` (WebAudio; no audio files);
  global mute in `src/state/settings.ts`.
- Offline via `vite-plugin-pwa` (`vite.config.ts`) — **anything new that must
  work offline has to be precached** (bundled, not fetched at runtime).

## Guiding principles (unchanged from lt-01)

1. **Offline-first.** New assets (audio clips, images) must be bundled and
   precached, or degrade gracefully when a runtime API (TTS, network) is absent.
2. **Minimal dependencies.** Prefer platform APIs (Web Speech, emoji, CSS
   `prefers-color-scheme`) over libraries.
3. **Forgiving & kid-friendly.** Additions are optional aids, never blockers.
4. **Data-driven.** Pronunciation and image live as data on `ContentItem`, so
   parent-added content participates too.
5. **Testable core.** Keep new logic (pronunciation mapping, theme resolution)
   pure and unit-tested; verify UI/rendering in the browser + on device.

## Suggested sequencing

Independent tracks; can land in any order. Rough size in parentheses.

1. **Dark mode** (S–M) — self-contained, high daily value, unblocks screenshots
   in both themes. Do first.
2. **Audio pronunciation, phase 1** (M) — Web Speech (nl-NL) for numbers and
   words + letter *names*; speaker button in the trace HUD; a "spraak" setting.
3. **Word images via emoji** (S–M) — `image` field, base-word emoji map, editor
   emoji picker, faint background render + reveal-on-complete.
4. **Audio pronunciation, phase 2** (M–L, content task) — bundled clips for
   correct letter *klanken* and consistent offline base-word audio.
5. **Word images: parent upload** (S) — optional, store a resized image in
   IndexedDB.

## Cross-cutting decisions to confirm

- **Letter audio: klank ("mmm") or letternaam ("em")?** Pedagogically the klank
  matters most for early reading, but TTS only says names. This drives whether
  phase 2 (bundled clips) is required. See `01`.
- **Word images: emoji vs illustrations vs uploads?** Emoji is the offline,
  zero-asset default; see `02`.
- **Dark mode: follow system only, or also a manual toggle?** Recommend system
  default + optional override in the parent settings. See `03`.
