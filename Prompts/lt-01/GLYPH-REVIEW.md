# Glyph Review Checklist

Tracks the pedagogical review of the base stroke data
(`src/data/base-content.json`, generated by `scripts/build-glyphs.ts`).

## Status legend

- ✅ machine-verified traceable + visually recognizable (author confidence)
- 👀 needs a **Dutch teacher / Pennenstreken** review of start point, direction
  (schrijfrichting), and stroke order
- ⚠️ shape or convention flagged for refinement

**Every glyph is currently 👀** — the shapes were authored directly with sensible
schrijfrichting and machine-verified as traceable, but none has yet been checked
against the actual Pennenstreken (Zwijsen) convention. Marking a row ✅ means a
Dutch reviewer has confirmed start/direction/order.

## How to review

1. Run the app on a glyph with the authoring overlay:
   `…/letter-tracer/?char=a&debug=1` (green dot = start + order number,
   red arrow = direction).
2. For each stroke check: correct **start point**, correct **direction**, correct
   **order** relative to other strokes.
3. To correct: edit the glyph in `scripts/build-glyphs.ts`
   (reverse a stroke = reverse its point run; reorder strokes = reorder the
   array), then `npm run build:glyphs` and re-check. `npm test` guards that every
   glyph stays traceable.

## Machine verification (automated, in CI)

- `src/data/base-content.test.ts` asserts the pack parses with **no warnings**,
  has all **65** items, and that **every glyph traces to completion** along its
  ideal path. This catches degenerate/broken data but says nothing about
  pedagogy.

## Author notes / open items (⚠️)

- **s / S** — the double-curve is a little lumpy; recognizable but a candidate
  for a cleaner two-arc shape.
- **r** — the arch is minimal; confirm it reads as an `r` at small sizes.
- **a** — single-story "ronde a" (bowl + right stem). Confirm this matches the
  Pennenstreken model (start at the top of the bowl, sweep left).
- **i / j** — dot is authored as the **second** stroke (stem first). Confirm the
  method's preferred order.
- **f / t / g / y / j** — ascender/descender extents and the exact hook/crossbar
  positions are approximate.
- Capitals **G, Q, J** — leg/tail/hook positions are approximate.
